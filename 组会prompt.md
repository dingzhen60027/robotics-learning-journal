# Prompt

## 本周工作

你是一名从事多机器人协同强化学习研究的研究生，需要撰写一份每周工作汇报。

请根据我提供的工作内容，按照以下结构生成周报：

要求：

1. 语言务实，不浮夸，不写空话。
2. 说清楚“具体做了什么”，避免泛泛而谈。
3. 每个部分内容点数不要过多（2-3点即可）。
4. 技术表达准确，但不要写成论文。
5. 风格偏科研组内汇报，简洁清晰。

结构如下：

师兄，我本周工作如下：

一、本周主要工作
（概括本周核心技术工作，写2-3点，说明具体做了什么、解决了什么问题）

二、取得成果与发现的问题
• 成果：
（写实际打通的流程、复现结果、性能表现等）
• 存在的问题：
（写当前暴露出的技术问题或不稳定点）

三、下周计划
（写2点即可，必须具体，例如测试什么、优化什么、分析什么，不要写“继续深入研究”这种空话）

我的本周实际工作内容如下：
[在这里填写你本周做的具体事情]

## PPT

你是一名研究生，需要准备一份组会汇报PPT。请根据我提供的“本周实际工作内容”，生成一套结构清晰、适合做PPT的页面方案。

整体要求：

1. 风格务实、自然表达，不浮夸，不写空话。
2. 说清楚：本周做了什么、做到哪一步、目前情况如何。
3. 不强行套“完成…解决…”句式，只要表达清楚即可。
4. 每页以条列式为主，避免大段文字。
5. 每页控制在3–6条要点，每条尽量不超过20字。
6. 每页必须包含：
   - 【页标题】
   - 【页面文字】
   - 【插图/图表建议】

建议页数：4-5页。

---

第1页：本周工作概览（只列2-3点，不展开细节）

要求：

- 用2-3条概括本周最核心的工作（工作A 工作B ...)
- 每条用1–2行说明主要进展
- 不展开技术细节
- 不写泛泛总结

插图建议：

- 可放一张整体流程图 / 工作结构示意图
- 或不放图，保持简洁

---

第2页：工作A —— 做了什么

内容：

- 本周围绕什么问题展开
- 做了哪些具体事情（实现/修改/对接/测试/整理等）
- 工作推进到哪一步

插图建议：

- 流程图 / 模块结构图 / 关键代码流程示意
- 或相关运行截图

---

第3页：工作A —— 当前结果与现象

内容：

- 当前状态（已跑通/已验证/已完成初测等）
- 观察到的结果或现象
- 是否存在波动或异常（客观描述）

插图建议：

- 曲线图 / 表格 / 实验截图
- 优先用图说明结果

---

第4页：工作B —— 做了什么

内容：

- 目标是什么
- 本周推进了哪些关键步骤
- 当前完成到哪一步

插图建议：

- 对比图 / 框图 / 截图
- 若内容偏整理分析，可用表格

---

第5页：存在的问题

内容要求：

- 列出2–3个关键问题
- 每个问题包含：
  - 现象（具体）
  - 初步判断原因（可写“可能”）

禁止：

- 空泛问题（如“效果一般”）
- 没有具体表现的描述

插图建议：

- 局部曲线放大图
- 失败案例截图
- 或简单示意图

---

第6页：下周计划（只写2点）

要求：

- 每点必须具体
- 说明“打算做什么 + 怎么做”
- 不写“继续优化”“持续推进”等空话

示例风格（不要固定套用）：

- 分析XX曲线，定位波动来源
- 调整XX结构，并重新测试对比

插图建议：

- 可不放图
- 或简单列出里程碑节点

---
